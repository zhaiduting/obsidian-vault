Window 应用程序只能使用 Alt / Ctrl / Shift 这 3 个物理修饰键，不能使用 Win 徽标键（系统占用），也不能识别苹果键盘的 Cmd 键。折中的办法是「将 Cmd 键映射成 Alt + Ctrl 的组合键」，这可以继续保持 Cmd 作为修饰键的功能。

### 安装 Mac 系统的按键映射工具

```sh
brew install --cask karabiner-elements
```

### 添加 Complex Modifications 规则

此规则的意思就是仅在 Moonlight 中将左 Cmd 键映射成 Ctrl + Alt 的组合

```json
{
  "description": "Moonlight: Left Command -> Ctrl + Alt",
  "manipulators": [
    {
      "conditions": [
        {
          "bundle_identifiers": ["^com\\.moonlight-stream\\.Moonlight$"],
          "type": "frontmost_application_if"
        }
      ],
      "from": {
        "key_code": "left_command",
        "modifiers": { "optional": ["any"] }
      },
      "to": [
        {
          "key_code": "left_control",
          "modifiers": ["left_option"]
        }
      ],
      "type": "basic"
    }
  ]
}
```

### 与 LinearMouse 配合使用

前文已述 LinearMouse 可以解决「Shift + 滚轮」问题，此功能对于 Karabiner Elements 同样有用，因为这可以确保「Shift + Cmd + 滚轮事件」绑定新的技能！

如果关闭 LinearMouse 则只能绑定「Cmd + 滚轮事件」，而不能绑定「Shift + Cmd + 滚轮事件」

### 在 WOW 中使用 Mac 的操作习惯

「将 Cmd 键映射成 Alt + Ctrl 的组合键」的另一个好处是可以在《魔兽世界》的文本框里使用 Mac 系统中复制、粘贴的操作习惯，这为魔兽宏命令的制作过程提供了些许便捷。在 Moonlight 中实际测试一下就知道了：

- Cmd + shift + v 可以将 Mac 里复制的文本直接粘贴到魔兽宏命令的文本框中
- Cmd + a 可以选中现有宏的所有文本
- Cmd + c 复制已选的文本
- Cmd + v 粘贴到新宏的文本框里

注意，以上快捷键对于 Windows 的文本编辑器无效，只能在魔兽世界的文本框里使用！

### 关于魔兽世界本身的 Bug

「Shift + Cmd + 滚轮下滚」可以绑定快捷键，但是却无法施放法术。这并非我们的 Cmd 键映射有问题，而是因为魔兽世界本身的 Bug 导致的：「Alt + Shift + Ctrl + 滚轮下滚」可以绑定快捷键，但就是不能施法。

奇怪的是：「Alt + Shift + Ctrl + 滚轮向上滚」却可以正常施法！

早在 2019 年 3 月份就有老外提交了这份 [Bug 报告了](https://us.forums.blizzard.com/en/wow/t/unable-to-use-altshiftctrlmouse-wheel-down-as-a-bind/126843?utm_source=chatgpt.com)，没想到现今 2026 年依旧存在。
