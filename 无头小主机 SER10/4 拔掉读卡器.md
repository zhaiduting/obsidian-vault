在 Sunshine + Moonlight 投屏成功的状态下可以关闭 OBS 程序，但却不能拔掉读卡器！解决方法：在小主机里安装虚拟显示器的驱动程序。命令如下

```sh
winget install "Virtual Display Driver"
```

之所以要安装这玩意，是因为 Window 在没有显示器的情况下不会输出图形界面。使用 winget 而不使用 choco 的原因是 choco 并未收录这个包。

事实上执行完 `winget` 的命令后 VDD 仍然无法正常工作，此时手动打开文件夹 `C:\Users\zhaid\AppData\Local\Microsoft\WinGet\Packages` 里面虚拟显卡驱动相关的子文件夹，然后双击其中的 `VDD Control.exe` 就会显示一个窗口

![VDD](https://lib.zhaiduting.work.gd/uPic/VDD.jpg)

再点击 VDD 控制界面的「Install Driver」一次，之后就可以正常拔掉读卡器了。此时的小主机与笔记本之间使用的是无线连接，真正实现了无头运行状态。
![VDD|](https://lib.zhaiduting.work.gd/uPic/VDD.png)

### 自定义分辨率

安装后的 VDD 默认只会读取最基础的几组分辨率，通过修改
`C:\VirtualDisplayDriver\vdd_settings.xml`
可以实现分辨率的自定义。试用以下内容**完全覆盖**你原来的 `vdd_settings.xml` 文件：

```xml

    <monitors>
        <count>1</count>
    </monitors>
    <gpu>
        <friendlyname>default</friendlyname>
    </gpu>
    <global>
        <g_refresh_rate>60</g_refresh_rate>
        <g_refresh_rate>90</g_refresh_rate>
        <g_refresh_rate>120</g_refresh_rate>
        <g_refresh_rate>144</g_refresh_rate>
        <g_refresh_rate>165</g_refresh_rate>
        <g_refresh_rate>244</g_refresh_rate>
    </global>
    <resolutions>
	    <!-- 全屏模式 -->
        <resolution>
            <width>3024</width>
            <height>1964</height>
        </resolution>
        <resolution>
            <width>3024</width>
            <height>1890</height>
        </resolution>
        <resolution>
            <width>1512</width>
            <height>982</height>
        </resolution>
        <resolution>
            <width>1512</width>
            <height>945</height>

            <width>1280</width>

            <width>1366</width>

        </resolution>
        <resolution>

            <height>768</height>
```

> 修改 vdd_settings.xml 文件后需要重启一下无头小主机，之后才能在「显示设置 » 显示器分辨率」菜单中更改分辨率。

### 最佳配置清单

为了在全屏避开刘海的同时，获得媲美 14 寸 Mac 原生的点对点 Retina 级清晰度，请将两端参数对齐如下：

| **配置项目**      | **Windows 小主机端设置**                                               | **Mac Moonlight 客户端设置**                  |
| ----------------- | ---------------------------------------------------------------------- | --------------------------------------------- |
| **画面分辨率**    | 桌面右键 $\rightarrow$ 显示设置 $\rightarrow$ 切换为 **`3024 × 1890`** | 选择 **`原生（不含刻痕）(3024x1890)`**        |
| **系统缩放**      | 调整为 **`200%`** （防止 Windows 字太小）                              | —                                             |
| **刷新率 / 帧率** | 设置为 **`120 Hz`**（匹配 Mac ProMotion）                              | 设置为 **`120 FPS`**                          |
| **视频编码器**    | —                                                                      | 强制 **`HEVC (H.265)`** 或 **`AV1`**          |
| **色彩格式**      | Sunshine 网页端设置 `Chroma Subsampling`: **`4:4:4`**                  | 勾选 **`强制 4:4:4 色彩`** 或 `10-bit` 高色深 |
| **串流码率**      | —                                                                      | 推荐设定在 **`80 ~ 100 Mbps`**                |

> [!TIP]
> 如果最佳配置导致界面卡顿或者主机发烫，可以试试将 Moonlight 刷新率降到 60 Hz，串流码率降到 60 Mbps，色彩格式改回之前的默认值。

> [!NOTE]
> 有些顽固的老程序对 200% 的系统缩放率视而不见，这会导致 3024 \* 1890 的分辨率下显示的字体太小。为此，不得不手动更改分辨率为 1512 \* 945 了。
>
> 还好，AI 给了我一个更高明的解决方法：《应用级 DPI 强制缩放》

### 哪些参数会导致小主机发热？

零刻 SER10 采用了高性能移动端处理器，串流时发热的核心源头是 **显卡的硬编码器（NVENC/VCE/QuickSync）高负载运转** 以及 **游戏本身的图形渲染压力**。以下参数是发热的“罪魁祸首”：

1. **分辨率（3024x1890）与游戏内分辨率**：
   - **原因**：3024x1890 的总像素点高达约 570 万，几乎是标准 1080P（200 万像素）的 **3 倍**。如果游戏内也开到了 3K 分辨率，小主机的 GPU 渲染压力会暴增，导致发热。
   - **优化建议**：如果是玩 3A 游戏，可以在 Windows 桌面保持 `3024x1890`，但在**游戏内将分辨率降低为 `1920x1200`（同样是 16:10 比例，画面不会拉伸变形）**，并开启游戏内的 DLSS 或 FSR 缩放。这样既能减轻 GPU 渲染发热，又不会破坏窗口比例。

2. **串流帧率（120 FPS）**：
   - **原因**：从 60 FPS 提升到 120 FPS，意味着小主机的显卡编码芯片在同一时间内**处理的画面帧数直接翻倍**。每秒要压缩 120 张 3K 分辨率的图片，编码器的发热会非常明显。

3. **色彩格式（4:4:4 编码）**：
   - **原因**：4:4:4 意味着不进行颜色抽样压缩，每一帧都保留最完整的色彩像素送去编码。这会导致编码芯片的工作量显著上升（尤其是 3K+120Hz+4:4:4 同时开启时），是诱发小主机风扇狂转的主要原因。

### 哪些参数会导致游戏出现马赛克（糊块/噪点）？

在串流中，马赛克的本质是**网络传输速度跟不上画面变化的剧烈程度**，或者**编码器在瞬间超载时为了保延迟而主动降低了画质**。

1. **码率（Bitrate）过低（致命原因）**：
   - **原因**：在 3024x1890 @ 120Hz 如此庞大的数据量下，如果 Moonlight 的码率还保留在默认的 `20~30 Mbps`，当游戏画面剧烈晃动（如赛车、转动视角、粒子特效）时，带宽会瞬间被撑爆，画面就会分裂成一格一格的马赛克。
   - **优化建议**：在局域网下，**务必将码率拉高到 `80 Mbps` 或更高**。

2. **视频编码器选错（H.264）**：
   - **原因**：如果在 Moonlight 设置里不小心选成了过时的 `H.264`，它的压缩效率很低，在 3K 高分下极易在高动态场景下崩溃出马赛克。
   - **优化建议**：必须选择 `HEVC (H.265)` 或 `AV1`。AV1 和 H.265 在相同码率下能提供成倍的画质细腻度，几乎不会产生糊块。

3. **局域网抖动 / Wi-Fi 信号干扰**：
   - **原因**：3K+120Hz 需要极其稳定的数据洪流。如果你的 Mac 使用了不稳定的 Wi-Fi 频段（如与蓝牙冲突的 2.4GHz 频段，或距离路由器太远），导致 Moonlight 出现瞬间丢包，Moonlight 为了维持画面不卡顿，会强制把接收到的残缺视频信号降画质拉伸，表现出来就是大面积的绿色或彩色马赛克。
   - **优化建议**：确保 Mac 连接的是路由器的 **5GHz 或 6GHz (Wi-Fi 6E/7)** 频段，并避开周围墙体的严重遮挡。
