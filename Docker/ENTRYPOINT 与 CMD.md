Dockerfile 文件内容如下

```sh
# 使用一个基础的 Linux 镜像
FROM alpine:latest

# ENTRYPOINT: 设定要执行的程序 (echo) 和一个固定参数 (-e)
# "-e" 是 echo 的一个参数，表示启用反斜杠转义 (例如 \n 换行)。
# 这个参数是固定的，不可被运行时参数替换。
ENTRYPOINT ["echo", "-e", "固定前缀:"]

# CMD: 设定默认的可变参数
# 这个参数会在 ENTRYPOINT 的固定参数之后被追加，并且可以被 docker run 时的参数覆盖。
CMD ["默认消息（可被覆盖）"]
```

### 构建为 hello 镜像

```sh
docker image build -t hello .
```

### 敲入两条命令测试一下

```sh
~ > docker run --rm hello
固定前缀: 默认消息（可被覆盖）
~ >
~ > docker run --rm hello "这是我替换的参数，它将取代默认消息"
固定前缀: 这是我替换的参数，它将取代默认消息
```

### 其实就是元素追加

CMD 方括号里的所有元素，直接追加到 ENTRYPOINT 方括号里。
以第1条命令为例，追加后变成下面这样
`["echo", "-e", "固定前缀:", "默认消息（可被覆盖）"]`
最终命令为
`echo -e "固定前缀:" "默认消息（可被覆盖）"`

第2条执行替换，然后追加，变成
`["echo", "-e", "固定前缀:", "这是我替换的参数，它将取代默认消息"]`
最终命令
`echo -e "固定前缀:" "这是我替换的参数，它将取代默认消息"`
