假设您**双击 Terminal.app/iTerm2 启动一个新的窗口**（通常是 Login Shell）：

| **步骤** | **文件**    | **作用**                                                                                                                                                             |
| -------- | ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1**    | `.zshenv`   | **最先加载**。添加 `$HOME/.cargo/bin` 到 `$PATH`。                                                                                                                   |
| **2**    | `.zprofile` | 接下来加载。执行任何登录时才需要的配置。                                                                                                                             |
| **3**    | `.zshrc`    | 接下来加载。配置别名、函数和提示符等交互式设置。                                                                                                                     |
| **4**    | `.profile`  | **通常会被跳过**，因为它只在 `.zlogin` 不存在且 `.zprofile` 也不存在时才作为 Login Shell 的最后兼容选项。如果您的 `.zprofile` 存在，那么 `.profile` **不会被加载**。 |

**简而言之：**

- **`.zshenv`**：确保 **Rust 环境变量** 在**所有** Zsh 实例中都可用。
- **`.profile`**：作为**兼容性后备**，以防在某些特殊的 Login Shell 启动场景中，`.zshenv` 或 `.zprofile` 未按预期工作。
- **`.zshrc`**：负责您日常使用的 **命令别名和 Shell 交互**。
