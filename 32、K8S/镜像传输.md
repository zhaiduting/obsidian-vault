将宿主机拉取过的镜像传输到集群内部，避免 kubectl 重新拉取

```sh
kind load docker-image alpine:latest --name dev-cluster
```

命令中的 dev-cluster 是集群名字，它是在创建集群时通过 `kind create cluster --name dev-cluster` 指定的。

如果创建集群时只运行了 `kind create cluster` 而没有指定集群名称，`kind` 会给它一个默认名字：`kind`。 对应的容器名就是 `kind-control-plane`

```sh
~ > kind get clusters
enabling experimental podman provider
dev-cluster # 集群名字

~ > podman exec -it dev-cluster-control-plane crictl images | grep alpine
~ > kind load docker-image alpine:latest --name dev-cluster
enabling experimental podman provider
Image: "alpine:latest" with ID "e8f9ca9f1870bc194d961e259fd1340c641bf188e0d02e58b86b86445a4bc128" not yet present on node "dev-cluster-control-plane", loading...
~ >
~ > podman exec -it dev-cluster-control-plane crictl images | grep alpine
docker.io/library/alpine                        latest                                  e8f9ca9f1870b       8.98MB
~ >
```
