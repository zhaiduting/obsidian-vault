跨平台开发不要使用 `println` 宏，要用 Dioxus (Tracing) 官方推荐的 5 个宏。按照“严重程度”由低到高排列如下：

| **宏**       | **对应颜色 (浏览器/终端)** | **适用场景**                                                            |
| ------------ | -------------------------- | ----------------------------------------------------------------------- |
| **`trace!`** | 灰色/细体                  | **最细碎的跟踪**。比如记录鼠标每一像素的移动，或者循环中每一次迭代。    |
| **`debug!`** | 蓝色/紫色                  | **开发调试**。记录变量的中间状态、函数进入/退出。                       |
| **`info!`**  | 蓝色/绿色 (最常用)         | **关键节点**。应用启动成功、用户完成登录、点击了某个重要按钮。          |
| **`warn!`**  | 黄色                       | **警告但不报错**。比如 API 响应太慢、检测到非预期的配置、组件渲染异常。 |
| **`error!`** | 红色                       | **严重错误**。API 请求彻底失败、程序 Panic、数据损坏。                  |

### 安装依赖

```
cargo add dioxus-logger
```

### 导入依赖

只要文件开头存在以下代码就足够了

```rust
use dioxus::prelude::*;
```

不需要显示导入 tracing 模块

```rust
// use tracing::info;
// use dioxus::logger::tracing::{Level, info};
```

这是因为 Dioxus 的 `prelude`（预导入）做得非常霸道。只要代码顶部导入了 `dioxus::prelude::*` 就会自动包含 **`info!`, `warn!`, `error!` 等宏**

### 初始化

主函数里执行 `initialize_default()` 之后调用 5 个宏，可以在浏览器的控制台看到输出信息。如果不执行初始化也是可以正常调用 5 个宏的，此时只能在 IDE 终端看到输出信息

```rust
fn main() {
    dioxus::logger::initialize_default(); // 可删除本行 ⚡️
    launch(App);
}
```

### 五大宏的使用场景深度示例

#### 🔴 `error!` (灾难级)

用于那些**必须立刻修复**，或者导致功能完全不可用的地方。

- **场景**：API 请求失败、无法读取本地保存的用户配置。
- **代码**：
  ```Rust
  if let Err(e) = response {
      error!("无法从服务器获取数据，页面将显示空白: {:?}", e);
  }
  ```

#### 🟡 `warn!` (警示级)

用于**非预期但程序还能跑**的情况，或者是对性能有潜在影响的操作。

- **场景**：用户输入了非法字符（已被前端拦截）、组件重新渲染次数过多。
- **代码**：
  ```Rust
  warn!("用户尝试上传的文件过大 ({} MB)，已自动拒绝", file_size);
  ```

#### 🔵 `info!` (业务逻辑级 - 最常用)

用于记录**程序的生命周期**和**用户的关键行为**。

- **场景**：组件挂载完成、用户点击登录、数据加载成功。
- **代码**：
  ```Rust
  use_effect(move || {
      info!("首页组件挂载完成");
  });
  ```

#### 🟣 `debug!` (开发调试级)

用于**辅助排查问题**。这些信息对普通用户没用，但对你很有用。

- **场景**：打印复杂的中间变量、记录函数进入/退出的参数。
- **代码**：
  ```Rust
  debug!(current_state = ?state.read(), "正在计算排序结果");
  ```

#### ⚪ `trace!` (微观跟踪级)

用于**极其频繁**的操作。

- **场景**：监听鼠标移动坐标、记录 WebSocket 收到的每一条原始心跳包。
- **代码**：
  ```Rust
  trace!("收到 WebSocket 原始字节流: {} bytes", raw_buf.len());
  ```

### 一个非常实用的技巧：结构化字段

官方非常推荐你利用 `tracing` 的结构化特性，而不是拼接字符串。这样在支持的终端里可以过滤字段：

```Rust
// 不推荐 (类似 println!)
info!("用户 {} 购买了物品 {}", user_name, item_id);

// 官方推荐 (结构化)
info!(user = %user_name, item = %item_id, "订单处理成功");
```

- `%` 表示使用 `Display` 打印。
- `?` 表示使用 `Debug` 打印（比如打印结构体）。

### 💡 为什么 `dx` 官方推崇这一套？

因为当你执行 `dx serve` 时，如果未来 `dx` 工具升级，它能直接通过这些结构化字段在你的终端里实现**分类显示**（比如只看 Error，或者只看特定组件的日志）。这是简单的 `println!` 永远做不到的。
