终端敲入 `dx serve` 命令，默认情况下会显示一个链接，点击后可以在浏览器里查看效果。

### 浏览器模式 ⭐️

在 Mac 里，[使用台前调度分组 App](../Mac/使用台前调度分组%20App.md) 将 IDE 与浏览器拉进同一个组。然后适当 IDE 与浏览器窗口的大小，避免完全遮盖。

> [!TIP]
> 到这里就行了，下文内容可以不用看了

### 桌面模式

如果不想用浏览器，可以改用桌面 App 进行预览。只需修改 `Cargo.toml` 文件的 `features.default` 值，如下：

```toml
[features]
default = ["desktop"]
```

此时敲入 `dx serve` 命令将会打开 `Dioxus App` 窗口并置顶显示。

> [!CAUTION]
> 实测发现 dioxus 0.7.5 在桌面模式下的开发体验不佳，建议维持 `default = ["web"]`

### 桌面模式（取消置顶）

如果想要禁止 `Dioxus App` 窗口的置顶功能，只需在终端敲入以下命令一次就可以永久生效

```sh
dx config set always-on-top false
```

再次敲入 `dx serve` 命令，打开的 `Dioxus App` 窗口就不会置顶了。对于**非置顶状态**的窗口可以使用台前调度功能将此窗口与 IDE 拉进同一个组。

### 模式比较

结论：浏览器模式可获得最佳开发体验，不足之处是「浏览器窗口不能缩小到极致」。

桌面模式下 `Dioxus App` 可以缩小到极致，这是此模式的优点。但是此模式有个致命缺陷，纠正报错的代码后，`Dioxus App` 窗口会被强行关闭，然后重新打开。

### dx 命令汇总

```sh
# serve 支持热重载 ⭐️
dx serve
dx serve  --example  <EXAMPLE>

# run 不支持热重载
dx run
dx run --example <EXAMPLE>

# 取消窗口置顶
dx config set always-on-top false

# 显示当前项目的配置
dx config format-print

# 显示全局配置文件
cat ~/.dx/settings.toml
```

不要使用 `cargo run` 命令，因为它不支持热重载，也不会加载 dx config 的相关配置。开发 `Dioxus App` 应该始终使用 `dx serve` 或者 `dx run` 命令。

不要使用组合键 `⌃ + r` 或者 `⌃ + ⇧ + r` 运行项目，因为组合键相当于执行 `cargo run` 命令。

如果想运行带有 main 函数的 `example/foo.rs` 文件，应该敲入 `dx serve  --example foo` 命令，或者 `dx run  --example foo`
