在 k8s 工作节点内执行 `journalctl` 命令尝试监测 `containerd` 的日志变化，结果碰到错误提示： **"Insufficient watch descriptors available"**

原因是触碰到了 Linux 内核的一个资源限制：**`inotify` 监视器句柄数量达到了上限。**

```sh
~ > podman exec -it kind-worker sh
# journalctl -u containerd -f --since "1 second ago"
Insufficient watch descriptors available. Reverting to -n.
-- No entries --
#
```

解决方法：

- 退出 kind-worker 节点
- 查询到当前 `fs.inotify.max_user_instances` 数值为 128
- 将其修改为 512

```sh
# exit
~ > sysctl fs.inotify.max_user_instances
fs.inotify.max_user_instances = 128
~ >
~ > sudo sysctl fs.inotify.max_user_instances=512
[sudo] password for parallels:
fs.inotify.max_user_instances = 512
```

再次敲入 `journalctl -u containerd -f` 就可以正常进入监测状态了。
