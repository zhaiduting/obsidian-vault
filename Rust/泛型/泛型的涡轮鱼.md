在 Rust 社区中，**“涡轮鱼”（Turbofish）** 是一个既幽默又形象的非正式术语，专门指代这个符号：`::<>`。

当你需要在表达式中手动指定泛型参数时，它就会出现。比如： `std::any::type_name::<i32>()` 或者 `collect::<Vec<u8>>()`。

---

## 1. 为什么叫这个名字？

这个名字的由来非常直白：**因为它长得真的很像一条正在向前冲刺、带着小尾巴的鱼。**

- `::` 是鱼吐出的**气泡**或者**推进引擎**（涡轮）。
- `< >` 是鱼的**身体**（或者像鱼的双鳍）。
- 中间的文字则是鱼**肚子里的东西**。

这个术语最早由 Rust 核心团队的成员在开发邮件列表中使用，随后迅速走红，甚至成为了 Rust 官方文档和报错信息中默认的称呼。

---

## 2. 为什么非要这颗“鱼头” `::`？

正如你之前提到的 TypeScript，它不需要这颗“鱼头”。但 Rust 需要，这是为了解决 **“解析歧义”**。

如果没有这颗鱼头，编译器在解析一行代码时会非常痛苦：

```rust
// 如果没有鱼头，这行代码有两种解释：
let x = generic_func<A, B>(c);

// 解释 A (泛型调用)：
// 调用函数 generic_func，泛型参数是 A 和 B，传入参数 c。

// 解释 B (数学比较)：
// (generic_func < A) , (B > (c))
// 先判断函数名是否小于 A，得到布尔值；然后再处理 B 是否大于 c。
```

在 Rust 的设计哲学中，**解析器（Parser）不应该依赖类型信息来理解代码结构**。编译器在还没给 `A` 和 `B` 定性之前，必须一眼看出这行代码是在“比大小”还是在“传泛型”。

**`::` 的出现就是发令枪：** 只要看到 `::<`，编译器就知道：“别管比大小了，后面这一串全是类型！”

---

## 3. 什么时候必须“喂鱼”？

并不是所有泛型调用都要喂鱼，只有当**编译器无法推断出类型**时才需要。

### 必须喂鱼的经典场景：`collect`

`collect` 是 Rust 里最依赖涡轮鱼的方法，因为它太灵活了。

```rust
let words = "hello world".split_whitespace();

// ❌ 报错：编译器知道你要搜集，但不知道你是要搜集成 Vec, HashSet 还是别的。
// let vector = words.collect();

// ✅ 喂鱼：明确告诉它搜集成 Vec
let vector = words.collect::<Vec<&str>>();
```

## 4. 逃避“鱼”的技巧

如果你觉得涡轮鱼破坏了代码的美感，Rust 提供了一个优雅的替代方案：**类型注解（Type Annotation）**。

```rust
// 方案一：涡轮鱼（在右边喂鱼）
let v = "123".parse::<i32>().unwrap();

// 方案二：类型注解（在左边声明）
let v: i32 = "123".parse().unwrap();
```

在这两个方案中，Rust 程序员通常认为**方案二更优雅**，因为它让逻辑部分（右边）保持简洁。但在链式调用（比如 `.map(...).collect::<Vec<_>>().iter()...`）中，涡轮鱼是不可或缺的，因为你没法给中间过程加左侧声明。

---

## 总结

- **形象**：`::<>` 看起来像条鱼。
- **功能**：在表达式中明确指定泛型类型。
- **必要性**：为了消除解析歧义，防止编译器把泛型当成大于小于号。

## 与 TS 风格对比

| **特性**     | **TypeScript**             | **Rust**                               |
| ------------ | -------------------------- | -------------------------------------- |
| **定义时**   | `function convert<T>(...)` | `fn convert<T>(...)`                   |
| **调用时**   | `convert<Jpy>(amount)`     | `convert::<Jpy>(amount)`               |
| **解析逻辑** | 依赖复杂的上下文推导       | 符号驱动，无歧义（看到 `::` 必是泛型） |

原以为 TS 中「传递泛型参数」的写法已经够丑的了，哪晓得 Rust 里面的更丑，竟然整出个「涡轮鱼」
