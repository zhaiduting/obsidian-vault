> [!TIP]
> 等式约束必须与关联类型结合使用

在 Rust 的泛型语法中，`T: Trait<Item = u32>` 这种写法，**专门且仅用于**约束 Trait 内部定义的 `type`（即关联类型）。

- 如果一个 Trait **没有** `type` 定义，你就根本没有地方可以放那个 `=` 号。
- 如果一个 Trait 使用的是**普通泛型参数**（如 `trait Trait<T>`），你指定类型时直接写 `Trait<u32>`，中间是不加等号的。

**对比一下：**

| **特征类型**       | **定义方式**                     | **调用/约束方式**          | **是否有等号？**   |
| ------------------ | -------------------------------- | -------------------------- | ------------------ |
| **普通泛型 Trait** | `trait Container<T>`             | `T: Container<i32>`        | **无**（直接传参） |
| **关联类型 Trait** | `trait Container { type Item; }` | `T: Container<Item = i32>` |                    |

当你为某个结构体实现（`impl`）一个具有关联类型的 Trait 时，你必须告诉编译器关联的具体类型是什么。但这并不意味着你的结构体内部必须有一个字段叫这个类型。

```rust
trait Task {
    type Output; // Trait 定义了关联类型
}

struct Worker; // 结构体本身没有任何字段，也没定义 type

impl Task for Worker {
    type Output = String; // 实现时必须指定：对 Worker 来说，Output 是 String
}
```
