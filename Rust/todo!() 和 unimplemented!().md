在 Rust 中，`todo!()` 和 `unimplemented!()` 的功能几乎**完全相同**。它们都是用来标记尚未完成的代码片段的宏，并且在执行到该位置时都会触发 `panic!`。

不过，虽然它们在技术层面没有差异，但在**语义和使用意图**上是有区别的。

| **宏**                  | **语义 (Semantics)**      | **适用场景**                       |
| ---------------------- | ----------------------- | ------------------------------ |
| **`todo!()`**          | **“我待会再写”**             | 开发过程中占位，表示功能已规划，但还没来得及实现。      |
| **`unimplemented!()`** | **“我不打算实现”** (或者“还没实现”) | 明确告知协作者（或未来的自己），某个分支或功能目前不被支持。 |

它们都可以用于赋值语句
