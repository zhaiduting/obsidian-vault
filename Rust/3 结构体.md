在 Rust 中有以下三种结构体：

1. **普通结构体 (Named-field)：** 全部用 `{ }`，字段必须有名字。
2. **元组结构体 (Tuple-like)：** 全部用 `( )`，字段没有名字，只有类型。
3. **单元结构体 (Unit-like)：** 连括号都没有，仅用于标记类型。

你必须在三种结构体形式中“三选一”，不能搞“混搭”。

### 普通结构体

```rust
#[derive(Debug)]
#[allow(dead_code)]
struct Person {
    name: String,
    age: usize,
    is_active: bool,
}

fn main() {
    let jack = Person {
        name: String::from("Jack"),
        age: 18,
        is_active: true,
    };

    // 创建 Jill，只改变 name 和 age，其他字段 (is_active) 从 jack 复制
    let jill = Person {
        name: String::from("Jill"),
        age: 19,
        ..jack // 复制 jack 中未显式设置的所有字段
    };

    println!("{:?}", jack);
    println!("{:?}", jill);
}
```

输出如下

```sh
Person { name: "Jack", age: 18, is_active: true }
Person { name: "Jill", age: 19, is_active: true }
```

结构体的实例化过程貌似比较繁琐，不能省略字段名称。如下所示的精简写法是错误的

```rust
let jack=Person{ "Jack", 18 }; // ❌
```

打印结构体需要使用 `#[derive(Debug)]`，占位符用 `{:?}`  
`..jack` 位于最后，而非开头，这与 JS 刚好相反
