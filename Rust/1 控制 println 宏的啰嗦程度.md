## 占位符的几种形式

| **格式**              | **语法** | **效果**                                                          |
| --------------------- | -------- | ----------------------------------------------------------------- |
| **Debug**             | `{:?}`   | 紧凑模式，适合日志输出。                                          |
| **Pretty Debug**      | `{:#?}`  | **井号在前**。自动换行、缩进，适合控制台调试。                    |
| **Display**           | `{}`     | 用户友好的自定义输出。                                            |
| **Alternate Display** | `{:#}`   | **井号在前**。触发 `f.alternate()` 逻辑，输出更冗长的自定义信息。 |

**注意：** `#` 永远紧跟在 `:` 后面，它的作用是让输出从“挤在一起”变成“整齐排队”。

## `eprintln!` 的占位符

与此类似，在 `eprintln!` 宏里也是通过占位符控制错误信息的详尽程度的

| **占位符**            | **显示效果**                               | **啰嗦程度**    |
| --------------------- | ------------------------------------------ | --------------- |
| `println!("{}", e)`   | **只打印最顶层的错误描述。**               | 🟢 极简         |
| `println!("{:#}", e)` | **打印整个错误链（用冒号连接各层原因）。** | 🟡 推荐（平衡） |
| `println!("{:?}", e)` | **打印错误链 + 堆栈回溯 (如果开启)。**     | 🔴 调试专用     |

## 举个例子

为了让你一眼看出这些占位符的本质区别，我们构建一个包含**嵌套列表**的 `Task` 结构体。

这里我们将 `Display`（给用户看）和 `Debug`（给程序员看）完全分开实现。

---

### 示例代码：任务状态监控

```rust
use std::fmt;

// 1. 定义结构体并派生 Debug (用于 {:?} 和 {:#?})#[derive(Debug)]
struct Task {
    id: u32,
    name: String,
    sub_tasks: Vec<&'static str>,
}

// 2. 手动实现 Display (用于 {} 和 {:#})impl fmt::Display for Task {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        if f.alternate() {
            // {:#} 的逻辑：详细模式
            write!(f, "任务[{}] '{}' 包含 {} 个子项", self.id, self.name, self.sub_tasks.len())
        } else {
            // {} 的逻辑：简洁模式
            write!(f, "{}", self.name)
        }
    }
}

fn main() {
    let task = Task {
        id: 101,
        name: "系统升级".to_string(),
        sub_tasks: vec!["备份数据", "更新内核", "重启服务"],
    };

    println!("--- 1. Display {{}} (最精简) ---");
    println!("{}", task);

    println!("\n--- 2. Display {{:#}} (自定义详细) ---");
    println!("{:#}", task);

    println!("\n--- 3. Debug {{:?}} (程序员速览) ---");
    println!("{:?}", task);

    println!("\n--- 4. Debug {{:#?}} (程序员美化) ---");
    println!("{:#?}", task);
}
```

---

### 运行结果

```sh
--- 1. Display {} (最精简) ---
系统升级

--- 2. Display {:#} (自定义详细) ---
任务[101] '系统升级' 包含 3 个子项

--- 3. Debug {:?} (程序员速览) ---
Task { id: 101, name: "系统升级", sub_tasks: ["备份数据", "更新内核", "重启服务"] }

--- 4. Debug {:#?} (程序员美化) ---
Task {
    id: 101,
    name: "系统升级",
    sub_tasks: [
        "备份数据",
        "更新内核",
        "重启服务",
    ],
}
```

### 核心记忆点

**`#` 的双面性**：

- 在 `Debug` 里，它是**排版引擎**（加换行和缩进）。
- 在 `Display` 里，它是**逻辑开关**（通过 `f.alternate()` 让你自己写 `if/else`）。

**书写顺序**：记住 `:#` 永远是一对好搭档，`:` 引导格式化，`#` 是增强插件，必须紧随其后。
