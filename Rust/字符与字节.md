#### **字符字面量 (Character Literal) —— `char`**

- **本质**：它代表一个 **Unicode 标量值 (Unicode Scalar Value)**。
- **存储**：在内存中固定占用 **4 个字节 (32-bit)**。
- **范畴**：它不仅能表示 `+` 或 `A`，还能表示中文 `‘中’`、拉丁文 `‘é’` 甚至表情符号 `‘🤔’`。
- **你的代码现状**：你目前的 `code` 变量类型是 `Vec<char>`，这意味着你正在使用这种“宽字符”来存储 Brainfuck 指令。

#### **字节字面量 (Byte Literal) —— `b' '`**

- **本质**：它代表一个 **原始的 ASCII 字节**。
- **存储**：在内存中仅占用 **1 个字节 (8-bit)**。
- **范畴**：它只能表示 ASCII 编码范围内的字符（即数值在 `0` 到 `127` 之间的字符）。
- **关联性**：这正是你之前在 Python 中尝试 `ord('+')` 得到 `43` 时所对应的底层数据类型。在 Rust 中，`b'+'` 等同于数字 `43u8`。

#### 与 u8 对比

**字节字面量 (Byte Literal)** 在类型上完全等价于 **`u8` (无符号 8 位整数)**。由于它们完全等价，你甚至可以在 Rust 代码里做“混合运算”：

```rust
let val = b'+' + 7;
println!("{}", val); // 输出 50 (因为 43 + 7 = 50)
```

然而有些情况下使用字节字面量比使用 u8 更合适，因为字节字面量能显著提高代码的可读性：

```rust
match code[pc] {
    b'+' => { /* 增加当前单元格的值 */ }
    43u8 => { /* 这里的 43 是什么意思？维护代码的人可能需要查 ASCII 表 */ }
    _ => {}
}
```
