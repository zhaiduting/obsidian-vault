```rust
let results: Vec<Result<i32, ()>> = vec![1, 2, 3]
    .into_iter()
    .map(Result::Ok).collect();
```

## 1. 这段代码在干什么？

它的目标是：**将一个整数列表变成一个“成功结果”列表。**

- **`vec![1, 2, 3]`**: 创建一个包含 `1, 2, 3` 的数组。
- **`.into_iter()`**: 将数组转成迭代器。
- **`.map(Result::Ok)`**: 这是最核心的部分。`map` 通常接收一个闭包（比如 `|x| Ok(x)`），但这里直接把 `Result::Ok` 当作一个**构造函数**传了进去。它会把每个数字 `x` 包装成 `Ok(x)`。
- **`.collect()`**: 把迭代器重新收集成一个 `Vec`。

最终 `results` 的内容是：`[Ok(1), Ok(2), Ok(3)]`。

## 2. 重点：`Result::` 到底能不能省略？

**结论：在这种特定语法下，不能省略。**

如果你写成 `.map(Ok)`，编译器会报错。原因如下：

### 语义冲突

在 Rust 中，当你直接写 `Ok`（不带括号）时，编译器会寻找一个名叫 `Ok` 的变量、函数或常量。

- **`Ok(x)`** 是一个**表达式**，它是合法的，因为编译器知道你在调用一个预导入的枚举变体。
- **`.map(Ok)`**：这里的 `map` 需要一个“函数指针”或“闭包”。虽然 `Ok` 是预导入的，但它在语法上被视为一个**构造器**，而不是一个可以直接作为参数传递的“函数值”。

### 为什么加上 `Result::` 就行了？

当你写 `Result::Ok` 时，你是在明确引用这个枚举变体的**路径**。在 Rust 的类型系统中，枚举变体（如果是元组型的）可以隐式地作为一个函数来使用。

## 3. 两种等价的写法

如果你觉得 `Result::Ok` 看着别扭，最通俗、最常用的写法其实是使用**闭包**：

```rust
// 写法 A：使用全称路径
let results: Vec<Result<i32, ()>> = vec![1, 2, 3].into_iter()
    .map(Result::Ok)
    .collect();

// 写法 B：使用闭包（最推荐，最清晰）
let results: Vec<Result<i32, ()>> = vec![1, 2, 3].into_iter()
    .map(|x| Ok(x)) // 这里可以省略 Result::，因为 Ok(x) 是个完整的表达式
    .collect();
```

### 为什么推荐写法 B？

1. **更直观**：新手一眼就能看懂是在把 `x` 变成 `Ok(x)`。
2. **更灵活**：如果你想在包装的同时给数字加 1，闭包很方便：`|x| Ok(x + 1)`。
3. **省略心智负担**：你不需要纠结 `Result::` 到底能不能省，直接写 `Ok(...)` 即可。
