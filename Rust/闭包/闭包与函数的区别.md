首先要了解 Rust 函数与 JavaScript 函数的区别。以下 JS 代码完全正确

```js
let s = "hello";
function foo() {
  console.log(s);
}
```

但是在 Rust 里却无法编译

```rust
let s = String::from("hello");
fn foo() {
	println!("{s}"); // ❌ 编译错误
}
```

这个对比这正好暴露了 JavaScript 和 Rust 在“函数是否能捕获外部变量”上的根本差异。

### 函数特点

Rust 里的 `fn`（函数项）不能捕获外部变量！⚠️

### 闭包救场

改成下面这样就对了，其中的 `||{...}` 表示一个无参数的闭包。在闭包里面可以正常访问闭包外面的变量。

```rust
let s = String::from("hello");
let bar = || {
    println!("{s}"); // ✅ 可以访问 s
};
```

### 结论

1️⃣ `fn`（函数项）特点：

- ❌ 不捕获环境
- ✅ 可以当函数指针用
- ✅ 零成本（没有额外结构）

2️⃣ closure（闭包）特点：

- ✅ 可以捕获环境
- ✅ 是一个 struct
- ✅ 有状态
