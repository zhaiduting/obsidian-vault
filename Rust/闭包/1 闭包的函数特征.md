[闭包的本质](1%20闭包的本质.md)中提到过，闭包相当于一个结构体的实例。根据变量的使用方式，此结构体会实现 `Fn / FnMut / FnOnce` 其中之一。

### 实现 Fn 的示例

```rust
let x = 10;
let f = || x + 1;
```

👉 捕获方式：`&x`

- `x` 被读取（只读）
- 没有修改
- 没有 move

等价结构体

```rust
struct Closure<'a> {
    x: &'a i32, // 指针
}

impl<'a> Fn() -> i32 for Closure<'a> {
    fn call(&self) -> i32 {
        *self.x + 1
    }
}
```

关键点

- `call(&self)`
- 变量只读

### 实现 FnMut 的示例

```rust
let mut x = 10;
let mut f = || {
    x += 1;
};
```

👉 捕获方式：`&mut x`

- `x` 被修改
- 必须用 `&mut x`

等价结构体

```rust
struct Closure<'a> {
    x: &'a mut i32, // 指针
}
impl<'a> FnMut() for Closure<'a> {
    fn call_mut(&mut self) {
        *self.x += 1;
    }
}
```

关键点

- `call_mut(&mut self)`
- 修改外部变量

注意，闭包变量也必须是：`let mut f = ...`
因为：👉 **调用 FnMut 需要 `&mut self`**

### 实现 FnOnce 的示例（重点）

```rust
let hello = "hello".to_string();
let world = "world".to_string();

let f = || {
    let borrow_hello = &hello;
    let move_world = world;
};
```

👉 变量 hello 的捕获方式：`&hello`
👉 变量 world 发生了 move！

- `world` 被移入闭包
- 并且在闭包执行时被**再次 move 到局部变量**
- world 最终从闭包中消费掉

这意味着闭包不可以重复调用，否则会因为 world 不存在而报错。等价结构体

```rust
struct Closure<'a> {
    hello: &'a String, // 指针
    world: String, // 不是指针，是实体 ⚠️
}
impl<'a> FnOnce() for Closure<'a> {
    fn call_once(self) {
        let borrow_hello = &self.hello;
        let move_world = self.world; // 移出 world 字段
    }
}
```

关键点

- `call_once(self)`
- struct 被“部分掏空”
- 不能再安全使用
- 闭包只能执行一次

### 变量捕获法则

捕获外部变量时，编译器在做类似这样的选择：

```rust
if 只读 {
    捕获 &T
} else if 需要修改 {
    捕获 &mut T
} else {
    捕获 T
}
```
