### 赋值语句

以下展示闭包在赋值语句中的写法

```rust
fn  add_one_v1   (x: u32) -> u32 { x + 1 }
let add_one_v2 = |x: u32| -> u32 { x + 1 };
let add_one_v3 = |x|             { x + 1 };
let add_one_v4 = |x|               x + 1  ;
```

只有第一行是函数，其余 3 个都是将闭包赋值给了变量。函数参数用 `()` 包裹，而闭包则用 `||` 包裹。闭包的参数类型及返回值类型，甚至花括号 `{}` 都是可省略的。

### 作为函数的参数传递

以下闭包 `|| self.get_default_greeting()` 没有参数，并且省略了类型注解，省略了花括号，作为 `unwrap_or_else` 的参数传递。

```rust
struct Store {
    name: String,
}

impl Store {
    fn greet(&self, user_name: Option<String>) -> String {
        user_name.unwrap_or_else(|| self.get_default_greeting())
    }

    fn get_default_greeting(&self) -> String {
        format!("Hello, {}!", self.name)
    }
}
```

### 捕获外部变量

这是闭包的重要功能。以上闭包捕获的 self 指的是调用 `greet` 方法的对象，它是 `Store` 结构体的实例。例如

```rust
fn main() {
    let store = Store {
        name: String::from("Rust Shop"),
    };

    // 传递用户提供的名字，或者使用默认的问候语
    let greeting = store.greet(None);
    println!("{}", greeting);
}
```

对于这个 main 函数中的 `store.greet(None)` 而言，定义在 `greet` 方法内的闭包捕获的 `self` 其实就是 `.greet` 之前的 `store` 对象。

不用闭包，改用函数捕获外部变量不行？对，不行！这是[闭包与函数的重要区别](0%20闭包与函数的区别.md)
