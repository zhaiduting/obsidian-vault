```rust
let mut animal: &dyn Speaker = &Dog;
```

上回笔记[胖指针和瘦指针](2%20胖指针和瘦指针.md)中 animal 是由 `data_ptr` 和 `vtable_ptr` 这两个指针组成的胖指针。数据指针指向 Dog 实例，虚表指针指向虚函数表 vtable

vtable 的**开头部分**存放的是有关类型的**元数据**，而 Trait 的方法地址则紧随其后。以下是 vtable 内存布局的“标准账本”：

| **偏移量 ** | **占用空间** | **存储内容**               | **作用**                                             |
| ----------- | ------------ | -------------------------- | ---------------------------------------------------- |
| **0**       | 8 字节       | **Drop 胶囊** (Destructor) | 指向 `drop_in_place`。告诉系统如何清理该类型的资源。 |
| **8**       | 8 字节       | **Size** (大小)            | 该具体类型（如 `Dog`）占用的字节数。                 |
| **16**      | 8 字节       | **Align** (对齐)           | 该类型的内存对齐要求（例如 8 字节对齐）。            |
| **24**      | 8 字节       | **第 1 个方法**            | 特征中定义的第一个方法的函数地址（如 `speak`）。     |
| **32**      | 8 字节       | **第 2 个方法**            | 特征中定义的第二个方法的函数地址（如 `sleep`）。     |

一个特定的 vtable 里面只有一种具体类型的信息。当你手里的胖指针 `animal` 指向 `Dog` 时，它的 `vtable_ptr` 指向的是专门为 `Dog` 生成的表；当你把它指向 `Cat` 时，这个指针会**整个跳走**，指向另一张专门为 `Cat` 生成的表。

## 1. 内存中存在多张表

编译器并不会把所有实现类都挤在一张大表里，而是为**每个“类型 + Trait”的组合**生成一张独立的、唯一的静态表。

- **VTable A**: 专门给 `Dog` 实现 `Speaker` 用的。
- **VTable B**: 专门给 `Cat` 实现 `Speaker` 用的。

---

## 2. 赋值时的“指针切换”

当你执行代码中的赋值操作时，发生的不是 vtable 内容的改变，而是胖指针内部 **`vtable_ptr` 地址的切换**。



```rust
let mut animal: &dyn Speaker = &Dog;
// 此时 animal 是: [ &Dog数据地址 | &DOG_VTABLE地址 ]

animal = &Cat;
// 此时 animal 变了: [ &Cat数据地址 | &CAT_VTABLE地址 ]
```

这就像你手里有一个带有两个插槽的插排：

1. 第一个插槽（数据指针）换了插头。
2. 第二个插槽（vtable 指针）换了另一本对应的说明书。
