如果你现在已经理解：

- trait object = 动态分发
- dyn = 标记动态分发
- vtable = 函数表
- `dyn Animal` 是 trait object 类型（DST，动态大小类型）
- `&dyn Animal` / `Box<dyn Animal>` 是实际使用的 trait object 形式

- `dyn Animal` = “接口本体（但没有大小）”
- `&dyn Animal` = “接口指针（真正能用）”

那下一步就可以真正进入 Rust 最难的部分了：

👉 **为什么有些 trait 不能写成 `dyn Trait`（object safety）**

这个会把你对 trait 的理解直接拉到“专家级”。
